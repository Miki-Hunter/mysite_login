项目名称： 非实时答题PK

基于Python django+web，vue等，前后端分离

面向对象: 青少年"娱乐"，闲暇百科知识补充

项目的想法：现在的很多人的碎片时间被短视频等占用，项目本意就是利用碎片时间“增益”，5分钟不到的时间快速了解15个各行各业方方面面的知识，包括但不限于天文地理、生物化学、政治军事、历史典故等，PK的形式会把写题本身的枯燥无味变得更有趣，积分的引入是把学习变成竞技游戏，增加玩家的竞争。考虑到用户数量以及用户时间，本项目主打“非实时”，即两个玩家在不同时空完成相同问题。此外，本项目本身还具有一定社交属性。

主要特点:
1)非实时
2)答题正确、申诉成功，积分奖励
3)积分等级对应权限
4)玩家可参与命题,可对答案、题解进行质疑申诉
5)暂定全部为单选题（竞技、自动判题） （前期用爬虫爬一些题目，数量起码要5位）


一些问题：
1）设置1，2，3三个难度等级，（考虑另外加一个没有上限的难度挑战EX）
2）为了负责，每一题都要配有题解
3）每一个玩家的每一次答题都要记录
4）核心：“非实时”  参考的是PDD模式
	“发起挑战”--> 新题目
	“接受挑战”-->做与上一个玩家相同的题目

对比后确定胜负
	胜者：.......
	败者：.......

5）答题模式：玩家可以自选一个难度，也可以选择“随机”，所以共5个选项
	暂定15题，答题时长上限5min，到点自动收题。
	（这里考虑EX级别要一定积分才能开启）

6）积分的设置（暂时没有想好具体设置，可能引入支付与提现）

7）答题需要一些“反馈”：（分A方、B方）
	A：每做一题，展示当前总耗时
	B：每做一题，展示与A的耗时差距

8）防止盲目刷分：
	设置每日答题上限，不同积分作出区分

9）关于出题：
	要有积分下限，然后等级越高，每天可以出的题目越多，
	（难度由出题玩家评定，因为别人做对了这个题目会得到相应积分，出题玩家要对题目负责，因为一旦有玩家申诉成功，出题玩家会受到一定处罚）
	EX难度提交后由管理员审核，其他由系统判断（只做查重工作）

10）错题复习：
	首先要求玩家数据的科学分类，（一人一库之类）
	暂时考虑一个偷懒做法：每次挑战的错题自动进入“错题库”，复习的时候写对了也不自动清理，由玩家自行清理

11）好题收藏：
	在每一个题目的界面都应该有这个选项，“收藏夹”可以设置可见

12）“实时PK”（plus功能）
	向在线玩家发出邀请，参与指定模式的PK
	（这意味着要做第3种“反馈”，以及实时聊天功能）
	当然了，不一定要“实时聊天”，做一个留言板，在大厅进行发言，然后只做一个发出和进入PK的命令，这样会节约得多，对方同意后，整个界面进入准备阶段（弹出提示），15s后开始PK。
